  10. Directives & There types
    1. In Angular, Directives are special markers on a DOM element (such as an attribute, element, or class) that tell Angular to perform specific actions or apply specific behavior to that element. Directives play a vital role in Angular, helping you to extend HTML capabilities and modify DOM behavior dynamically.
    2. There are three main types of directives in Angular:
      1. Component Directives (a special type of directive):-Components are essentially directives with their own templates. Every Angular component is a directive with a template.selector: The app-example selector can be used in HTML like to invoke this component.
      2. Structural Directives:-Structural directives alter the layout of the DOM by adding, removing, or manipulating elements. They change the structure of the view. These directives use the * symbol when applied in templates.
        1. *ngIf: Conditionally includes/excludes elements.
        2. *ngFor: Repeats an element for each item in a list.
        3. *ngSwitch: A set of directives to switch between different templates.
      3. Attribute Directives:-Attribute directives change the appearance or behavior of an element. Unlike structural directives, they do not modify the DOM structure but can manipulate DOM attributes.
        1. ngClass: Adds/removes a set of CSS classes.
        2. ngStyle: Adds/removes inline styles.

      How to set routing using module components(version 14 below)

    1. ng generate module admin --routing
    2. ng generate component admin/login
    3. ng generate component admin/profile
    4. ng generate component admin/data
    5. Define Routes in admin-routing.module.ts:- import { NgModule } from '@angular/core'; import { RouterModule, Routes } from '@angular/router'; import { LoginComponent } from './login/login.component'; import { ProfileComponent } from './profile/profile.component'; import { DataComponent } from './data/data.component'; const routes: Routes = [ { path: 'login', component: LoginComponent }, { path: 'profile', component: ProfileComponent }, { path: 'data', component: DataComponent }, ]; @NgModule({ imports: [RouterModule.forChild(routes)], exports: [RouterModule], }) export class AdminRoutingModule {}
    6. Define loadChildren in app-routing.module.ts :- import { NgModule } from '@angular/core'; import { RouterModule, Routes } from '@angular/router'; const routes: Routes = [ { path: 'admin', loadChildren: () => import('./admin/admin.module').then(m => m.AdminModule), }, ]; @NgModule({ imports: [RouterModule.forRoot(routes)], exports: [RouterModule], }) export class AppRoutingModule {}
    7. Must import AdminModule in app.module.ts:- import { AdminModule } from './admin/admin.module';
    8. Set routing in app.component.html:-

  17. Services:- Services is a class or piece of code that we reuse in more than one component to show data.Like we show the services data in header and footer also. For example:-We have student data that alrady show in Header component, so user also want to show user data in Footer component,
    1. Create UserdataService.ts
    2. Inside userdata.service.ts, define a list of users and a method to retrieve it:- import { Injectable } from '@angular/core'; @Injectable({ providedIn: 'root' }) export class UserdataService { private users = [ { name: 'Alice', email: 'alice@example.com' }, { name: 'Bob', email: 'bob@example.com' }, { name: 'Charlie', email: 'charlie@example.com' } ]; constructor() { } getUsers() { return this.users; } }
    3. Use UserdataService in AppComponent, import { Component, OnInit } from '@angular/core'; import { UserdataService } from './userdata.service'; @Component({ selector: 'app-root', templateUrl: './app.component.html', styleUrls: ['./app.component.css'] }) export class AppComponent implements OnInit { users = []; constructor(private userdataService: UserdataService) {} ngOnInit() { this.users = this.userdataService.getUsers(); } }
    4. Add UserdataService to providers in app.module.ts
    5. Display User List in app.component.html, Use *ngFor to iterate over the users and display their names and emails. You can add this at the top of the file or wherever you want the list to appear.
