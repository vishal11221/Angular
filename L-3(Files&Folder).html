Important Files and Folder Structure

1. package.json 2. node_modules 3. src Folder a.app b.assests c.enviornments d.index.html e.main.js f.style.css For Advance use:- a.package-lock.json b.angular.json c.tsconfig files d. .browserlistrc e. karma.config.js f. pollyfill.ts 1.package.json:- In package.json folder, your project name,your angular current version,your all commands are their which command help for which purpose. 2.node_modules:- In node_modules folder, your all libraries already install in this file that help to build your angular application, don't create any new file in this node_modules folder. 3.src folder:- a.src folder is most Important folder of your angular project,we use 99% of works in that folder. b.app folder contain piece of code that reuse again and again in code.It is component folder c.assests folder contain your extra css and extra images files. d.enviornments folder , which provides configuration for production,the default environment. e.favicon contain icon of your Angular icon , default icon f.index.html file is the main file of your Angular project without it your Angular project not started. html file. Angular is a framework which allows us to create "Single Page Applications", and here the index.html is the single page which was provided by the server. g.main.js file, main.js contains all our code including components (ts, html and css codes), pipes, directives, services and all other imported modules (including third party). h.style.css, in style.css we write our all global css code that apply in our project. i.package-lock.json,contain your all angular package details. j.angular.json, used for build and development tools provided by the Angular CLI. k.tsconfig.json, used to build an application, in which the types compiler option is set to [] to disable automatic inclusion of declarations from @types/* . it is our Application configuration. L.tsconfig.app.json , it is global configuration. m.tsconfig.spec.json, it is testing configuration file. n.browserlistrc, it contain all the browser list that support your angular application. o.karma.confs.js , it's like config files,like host no,password and all. 1. What happen when we delete package.json ? Then our Angular application not run.Because it has all the details of our application. 2. What is the difference between package.json and package-lock.json ? The “package.json” file defines the rules required to run your application and install dependencies. On the other hand, the “package-lock.json” file holds detailed information on all the dependencies installed based on the package.
